# Python Data Work

Prefer clear, reproducible code using the project's existing stack. For
dataframes, make copies intentionally, name intermediate results clearly,
and avoid chained assignment ambiguity.

Check shapes and schemas after reads, filters, joins, groupbys, pivots,
and writes. Validate aggregate totals before and after transformations.

When producing scripts, include a small main path, explicit inputs and
outputs, deterministic seeds when relevant, and readable errors for
missing files or malformed data.
