# Data Analysis

Start by understanding the data shape: columns, types, row counts,
missingness, duplicates, keys, time ranges, units, and obvious outliers.
Do not assume field meanings when names are ambiguous.

Keep analysis reproducible. Prefer code or SQL that can be rerun over
manual arithmetic. Validate totals and joins before interpreting results.

When presenting findings, separate observations from interpretation.
Include denominators, time windows, filters, and caveats. Use concise
tables for comparisons and charts when trends or distributions matter.
